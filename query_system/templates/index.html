    Neo4j Graph traversal

    Structural relationships — call paths, CFG reachability, dependency chains. Cypher shortestPath traverses dependency edges in milliseconds; SQL would need recursive CTEs.

    SQLite Quantitative metrics

    Coverage hit counts, test pass/fail rates, fuzz statistics. Aggregate queries (SUM, COUNT, GROUP BY) are SQLite's strength. No need for a graph for tabular coverage data.

    Q3 Cross-database join

    Query 3 joins both: Cypher finds functions reachable from main, then Python queries SQLite for each function's hit counts. Best of both worlds.

    Analysis Pipeline

    8-step automated pipeline from source code to queryable graph

    Pipeline Steps
    1
    Compile
    cmake .. -DCMAKE_CXX_COMPILER=clang++-14
    make plywood_calc plywood_calc_cov
    Debug build + coverage-instrumented build (--coverage)
    2
    Emit LLVM IR
    clang++-14 -S -emit-llvm -g -O0 plywood_calc.cpp
    880 instructions · debug metadata preserved (-g)
    3
    Call Graph Extraction
    opt-14 -enable-new-pm=0 -dot-callgraph plywood_calc.ll
    6 user functions · 6 call edges (stdlib filtered out)
    4
    CFG Extraction
    opt-14 -enable-new-pm=0 -dot-cfg plywood_calc.ll
    6 per-function DOT files · CFG blocks and edges
    5
    Data Dependency Extraction
    python3 dep_extractor.py plywood_calc.ll
    Parses DILocalVariable + llvm.dbg.declare for source-level names · — DEPENDS_ON edges
    6
    Clang Static Analyzer
    scan-build-14 clang++-14 -Wall -Wextra plywood_calc.cpp
    0 bugs found · 0 warnings after dead store removal
    7
    Test Execution + Coverage
    ./plywood_calc_cov <args> · bash stdin_tests.sh
    curated CLI + stdin tests · llvm-cov gcov · live coverage imported
    8
    Graph + Coverage Import
    python3 graph_importer.py --clear --callgraph ... --coverage ...
    Neo4j graph + SQLite coverage records refreshed from pipeline artifacts
